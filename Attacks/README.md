# Configs – ELK Stack & Sensor Configuration

[![ELK Stack](https://img.shields.io/badge/ELK-8.x-orange?logo=elastic)](https://www.elastic.co/)  
[![Filebeat](https://img.shields.io/badge/Filebeat-Log%20Shipper-blue)](https://www.elastic.co/beats/filebeat)

**Overview**  
This folder contains all configuration files and visual proof of the SIEM pipeline setup:  
- Elasticsearch (storage & indexing)  
- Logstash (log parsing & forwarding)  
- Kibana (dashboards & alerting)  
- Filebeat (log shipping from Snort on Ubuntu)  
- Supporting services (Java, rsyslog, fail2ban, Apache)  

All 14 screenshots are in **[./screenshots/](screenshots/)** — click any image name below to view full size.

### Complete Screenshot Documentation

1. **[1- Java Version.png](screenshots/1-%20Java%20Version.png)**  
   Verifies that OpenJDK 11 is correctly installed and active on the system.  
   This is a mandatory prerequisite for running Elasticsearch, Logstash, and Kibana.  
   Command used: `java -version` — output confirms build 11.0.28+.

2. **[2- ELK Status.png](screenshots/2-%20ELK%20Status.png)**  
   Shows the status of all three core ELK services using `systemctl`.  
   Confirms Elasticsearch, Logstash, and Kibana are all **active (running)** with no errors.  
   This is the first major milestone proving the stack is operational.

3. **[3- Kibana Status.png](screenshots/3-%20Kibana%20Status.png)**  
   Specific `systemctl status kibana.service` output.  
   Proves Kibana is loaded, enabled on boot, running with PID, and consuming expected memory/CPU.  
   No crashes or failed states — ready for GUI access.

4. **[4- Kibana.yml Config.png](screenshots/4-%20Kibana.yml%20Config.png)**  
   Key section of `/etc/kibana/kibana.yml` file.  
   Most important setting: `server.host: "0.0.0.0"` — allows Kibana to accept connections from any IP (remote access from other VMs).  
   Without this, Kibana would only be accessible from localhost.

5. **[5- Kibana GUI.png](screenshots/5-%20Kibana%20GUI.png)**  
   First successful load of the Kibana login page in a browser.  
   URL used: `http://192.168.67.130:5601` (or https if SSL configured).  
   Confirms Kibana web interface is reachable and ready for login.

6. **[6- ELK new value.png](screenshots/6-%20ELK%20new%20value.png)**  
   Output of the `elasticsearch-reset-password -u elastic` command.  
   Displays the newly generated password for the elastic superuser account.  
   This password is required for Logstash output, Kibana login, and API access.

7. **[7- Kibana Enrol Token.png](screenshots/7-%20Kibana%20Enrol%20Token.png)**  
   Enrollment token generated by Elasticsearch for secure Kibana connection.  
   Command used: `elasticsearch-create-enrollment-token -s kibana`.  
   Token is pasted into Kibana setup wizard to establish trust.

8. **[8- Kibana Verification Code.png](screenshots/8-%20Kibana%20Verification%20Code.png)**  
   Verification code shown during Kibana enrollment process.  
   This code is used in the browser wizard to complete the secure handshake.  
   Proves successful integration between Kibana and Elasticsearch.

9. **[9- Logstash Config.png](screenshots/9-%20Logstash%20Config.png)**  
   Main Logstash pipeline configuration file (likely `snort.conf`).  
   Shows input (beats port 5044), grok filter for Snort parsing, and output to Elasticsearch.  
   Critical piece for log ingestion and transformation.

10. **[10- Logstash status.png](screenshots/10-%20Logstash%20status.png)**  
    `systemctl status logstash` output.  
    Confirms Logstash is active, enabled on boot, running for some time, with PID and memory usage visible.  
    No startup errors — pipeline is healthy.

11. **[11- RSYSLOG Config file.png](screenshots/11-%20RSYSLOG%20Config%20file.png)**  
    rsyslog configuration file (e.g. `/etc/rsyslog.d/20-apache.conf`).  
    Shows forwarding of Apache access logs to Logstash using local6 facility and UDP/TCP port.  
    Enables web server logs to flow into the SIEM pipeline.

12. **[12- fail2Ban.config.png](screenshots/12-%20fail2Ban.config.png)**  
    fail2ban jail configuration example (e.g. for nmap or SSH).  
    Shows enabled jail, port monitored, log path, maxretry, bantime settings.  
    Adds automatic IP banning for repeated malicious activity.

13. **[13- Apache.config.png](screenshots/13-%20Apache.config.png)**  
    Apache configuration snippet (e.g. from apache2.conf or sites-enabled).  
    Confirms Apache is configured to serve web content and generate access logs.  
    Necessary for log collection and web probe detection.

14. **[14- Elastic index working.png](screenshots/14-%20Elastic%20index%20working.png)**  
    Proof that Elasticsearch index pattern (`snort-alerts-*` or `syslog-*`) is created and receiving data.  
    Visible in Kibana or via dev-tools — confirms logs are successfully indexed.
